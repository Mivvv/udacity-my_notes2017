Lesson 1: Learn To Code

Some Definitions

  • HTML: Hyper-text Markup Language
  • HTTP: Request protocol (Hyper-text transfer protocol)
  • W3Schools: HTML tag reference/documentation

HTML is the basic language of creating webpages. It provides base grounds for modern languages to work around. HTML forms this basis by "tags", they have simple stylistic properties; but also they are used to create the draft of the webpage by block elements. (explained later) To Summarize: HTML is simple styling of webpage, which also creates layouts for modern languages.

HTML Tags

As I've mentioned before, HTML coding is done by tags. Tags may look basic and they may serve simple purposes; but that doesn't change the fact that they are the key components of webpages. Here you will find some frequently used HTML tags and where & how they are used. Small definition: Anything with "<>" around is called a tag!

Input: <b> BOLD </b>
Output: BOLD

Using this tag allows your text to be shown in bold.
Input: <em> ITALIC </em>
Output: ITALIC

Using this tag allows your text to be shown in italic.
Input: <a href="Webpage-URL"> HYPERLINK-TEXT </a>
Output: HYPERLINK-TEXT

Above tag has two parts to use.
In href, we give the webpage url that we are trying to link.
In between <a></a>, we write the text that will be shown on the webpage as a link.
(Trivia: href means "Hypertext Reference")
Input: <img src="Image URL" alt="TEXT">
Output:

Above tag is a VOID tag, thus it doesn't need a closing tag.
Using it shows an image, and if image fails to load (as in here) it shows placeholder text

HTML Document Basis

<!DOCTYPE HTML>: Document type, helps browser rendering the correct document type.

<html>: Top level of HTML file, wraps all content.

<head>: Metadata, Title, CSS, Javascript etc. is found here.

<title> Title of page, displayed at the top of browser. </title>

</head> - end of head.


<body>: body of HTML, actual content is found here.

</body> - end of body

</html> -end of html

Lesson 2: Make A Stylish Webpage

Some Definitions

  • HTML: Hyper-text Markup Language
  • CSS: Cascading Style Sheets
  • DOM: Documentt Object Model

As previously I've said, HTML is creates layouts. The styling of those layouts are done by CSS codes. Remember when I said block elements will be important later. This is the later part. CSS codes allow you to stylize those block elements individually. To give a simple example, you can change the background color of that block, text color of its contents and even the blocks height and width. This leads us to another fact of webpage design. Everything is boxes, you should be thinking with boxes too. Now you are thinking with portals. It may be your own design, it may be given by a designer. What you need to do is GROUP elements into boxes. From large boxes to small boxes, then after boxifying the design you can create respective div's and classes.

Adding CSS

First things first, to make our life easier and design process without any repetitions, we will use div classes.
<div class="CLASS NAME"> </div>.
By doing so, whenever we style CLASS NAME, it will be applied to all div's with CLASS NAME. Thus saving us some precious time.

We can use CSS, inside the HTML file; but it would create a cluster of mess. To prevent looking at a tiring mess of codes, it is suggested to code CSS on a different file and link that file into HTML.
Into the <head> type:
<link rel="stylesheet" type="text/css" href="style.css">
Thus your HTML file will be looking for style.css inside the SAME FOLDER that HTML file is currently in and apply the styles found inside that file.

Input:
.CLASSNAME{
color: red;
}
Output:
Text inside classname.



Now lets move step by step by looking at this example.
".CLASSNAME" defines the class name for divs with that name. An example of Object-oriented programming.
Inside the curvy brackets, CSS codes are there which gives styling/attributes to the defined class.
For this example classname gives light-blue background color for the "text inside classname"

Also, there may be classes inside classes with different attributes. The smallest child class overrides overlapping rules.

Input:
<.CLASSNAME{
display: flex;
}
Output:
Multiple block elements inside one block element are arranged in a flexible manner.


What I mean by that can be learned by visiting this link.
But to simply put it, divisors create line breaks and blocks. Flex allows multiple divisors in a divisor to be arranged right next to each other rather than their default horizontal positioning. (It also has many more properties, just check the link)

Box Sizing and Positioning

First of all boxes has 4 elements.

  • Margin: creates blank space between the box and other elements on the webpage. This prevents overlapping items.
  • Border: creates a border around the padding and the content.
  • Padding: creates a blank space between border and the content.
  • Content: obvious isn't it?
Now, every element has a size and this may cause trouble when trying to create an exact size. Whenever you are resizing a block element, you must remember that there are 4 elements to take into consideration.
Other than that, it is a good practice to set size in terms of percentage rather than pixels. This will prevent boxes overlapping or going out of window border due to the size of the browser.

As in MDN webdocs; some experts recommend applying box-sizing: border-box to all elements.

Because of that, In lesson 2.4.11, we are given to use

* {
-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
-ms-box-sizing: border-box;
box-sizing: border-box;
}

in most of our CSS codes as a default.
